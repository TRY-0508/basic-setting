# 外部资源调研与文档规范

在项目推进过程中，需要调研外部资源（学术 paper、开源项目等）作为设计参照。本规范定义资源的组织方式和分析文档的编写标准。

## 1. 目录结构

资源统一存于 `resource/` 下，按类型和主题分层组织：

```
resource/
├── papers/                    # 学术论文
│   ├── INDEX.md               # 总索引（分类导航 + 数量统计）
│   ├── category-a/OVERVIEW.md # 分类总览
│   ├── category-a/*.pdf       # 该类论文 PDF
│   ├── category-b/OVERVIEW.md
│   └── ...                    # 按研究主题分为 N 个分类
│
└── projects/                  # 外部项目
    ├── INDEX.md               # 总索引（分类导航 + 数量统计）
    ├── type-a/                # 按类型分类
    │   ├── OVERVIEW.md        # 分类总览
    │   ├── project-a.md       # 独立项目分析
    │   └── project-b.md       # 同上
    └── type-b/
        ├── OVERVIEW.md
        └── ...
```

## 2. 论文分类总览文档规范

每个分类的 `OVERVIEW.md` 必须对每篇论文包含以下五个维度的详细分析：

| 维度 | 中文标识 | 内容要求 |
|------|---------|---------|
| Motivation | **出发点** | 论文要解决什么问题？现有方法为什么不够？目标的领域背景 |
| Innovation | **创新性** | 与现有方法相比的核心差异是什么？技术贡献的独特性在哪里？ |
| Approach | **做了什么** | 具体的技术方案、系统架构、关键算法、实现细节 |
| Validation | **怎么验证** | 实验设置、数据集、基线方法、具体数字（样本量、p 值、提升幅度）、消融实验、局限性 |
| Value | **价值** | 对本项目的具体启示——哪个模块、什么设计方向、可借鉴的具体模式 |

要求：
- "怎么验证"必须包含可引用的具体数字和实验设置，不能只有"验证了有效性"这样的笼统描述
- 每个维度 3-8 句话，信息密度要高，读者看完就能清楚知道这篇 paper 的核心思想和实验方法
- 所有实验数字直接来自原论文，不得伪造
- 文档使用中文编写

## 3. 项目分析文档规范

每个外部项目的独立 `.md` 文件必须包含以下四个维度的详细分析：

| 维度 | 中文标识 | 内容要求 |
|------|---------|---------|
| Motivation | **出发点** | 项目要解决什么问题？目标用户是谁？与现有方案的关键差异 |
| Innovation | **创新性** | 核心技术创新点、独特设计模式、与竞品的差异化 |
| Implementation | **实现内容** | 代码架构、关键组件、技术栈、规模指标（LOC、文件数、贡献者） |
| Value | **价值** | 对本项目的启示——验证了什么设计假设？挑战了什么设计假设？可借鉴什么具体模式？ |

要求：
- 头部信息必须包含：GitHub Stars、许可证、版本号、语言
- 尽可能包含 ASCII 架构图展示核心组件关系
- 必须有"对本项目的关键启示"小节，明确列出验证/挑战/可借鉴点
- 文档使用中文编写

## 4. 新增资源流程

1. **发现新论文/项目** → 判断分类归属
2. **论文**：
   - 更新对应分类的 `OVERVIEW.md`，按五维度格式添加条目
   - **下载 PDF**：论文 PDF 通常较小，应尝试从 arXiv 等开放平台下载（`https://arxiv.org/pdf/{ID}`），保存到对应分类子目录下。文件命名为 `{论文标题}_{Year}.pdf`（空格替换为连字符，如 `DEVS-Gen-Specification-Driven-Discrete-Event-World-Models_2026.pdf`），与总览文档中的论文标题一致
3. **项目**：在对应分类下创建独立 `.md` 文件，按四维度格式编写
4. **更新索引**：同步更新 `INDEX.md` 中的数量统计和导航
5. **交叉引用**：如果新内容与已有分类相关，在相关 OVERVIEW 中添加 `[see also]` 引用
